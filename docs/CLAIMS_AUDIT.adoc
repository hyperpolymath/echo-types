== Methodology

* One row per claim-bearing sentence. Restatements are kept as
  separate rows so the surface multiplicity is visible.
* `Surface` column: short token. `paper-front` = the `[IMPORTANT]`
  block at the top of `paper.adoc`. `paper-intro` = В§1 Introduction
  including the Contributions list. `abstract-*` = sections of
  `types-abstract.adoc`. `readme-*` = sections of `README.md`.
  `gh-desc` = the GitHub repository description.
* `Mapping` column: `module.theorem` or `module` when the claim is
  about a whole module / structural artefact. `NONE-PROPOSED` when
  no in-repo artefact corresponds. Multi-module mappings are joined
  with `+`.
* `Note` column: structural observations only вҖ” not judgements of
  whether the claim is justified. Each row is mapping-only per the
  brief.
* `VERDICT` + `EVIDENCE` columns added 2026-06-10 by source-only
  audit against the cited Agda under `--safe --without-K`.
  VERDICT vocabulary: `MATCHES` (theorem supports sentence as
  written) | `WEAKER-THAN-CLAIM` (theorem exists but is strictly
  weaker, e.g. pointwise vs strict, K-free skeleton vs full HoTT
  statement, or partial closure vs umbrella) | `NO-BACKING` (no
  theorem in the codebase backs the sentence) | `CONJECTURE-OK`
  (sentence is openly a non-claim, positioning, scope, retraction
  citation, or `NONE-PROPOSED` placeholder, and that is honest).
