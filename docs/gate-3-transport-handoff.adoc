== What this example demonstrates

The example exhibits structured loss from two distinguishable sources
on `Field n = Vec ‚Ñö n` over a periodic grid. (i) The explicit-Euler
diffusion step `step` at `Œª = ¬º` is *singular at the Nyquist mode* on
even grids: the alternating vector is annihilated. (ii) Block
averaging `avg` is non-injective (rung 2). (iii) Their composition
`avg ‚àò step` has a kernel that *strictly enlarges* beyond either
source taken alone. It is a worked instance of `EchoGraded`'s
base-Echo structure ‚Äî rung 4's `echoStep-is-Echo` shows the
transport-echo family is definitionally `Echo (avg ‚Ä¶)` ‚Äî and it
introduces no new bridge axis.

== Load-bearing theorems

* `step-kernel-nontrivial` ‚Äî explicit-Euler diffusion at `Œª = ¬º` on a
  periodic n-cycle is singular at the Nyquist mode for even `n`:
  an explicit nonzero field `f` with `step f ‚â° 0` at `n = 2`.
* `step-kernel-contains-nyquist-multiples` ‚Äî every scalar multiple of
  the Nyquist mode is annihilated:
  `‚àÄ c ‚Üí step (map (c *_) (nyquist 2)) ‚â° replicate 4 0‚Ñö`
  (containment, at the fixed even size).
* `nyquist-in-step-kernel` ‚Äî the `‚àÄ n` form:
  `‚àÄ n ‚Üí step (nyquist n) ‚â° replicate (2¬∑n) 0‚Ñö` (closes former gap
  (a), via the extracted `VecRotation` module).
* `step-kernel-at-4-is-nyquist-line` ‚Äî the rank-1 converse at n = 4:
  `step f ‚â° 0 ‚Üí Œ£[ c ] f ‚â° map (c *_) (nyquist 2)` (closes former
  gap (b), 4√ó4 ‚Ñö solve via the ring solver).
* `compose-kernel-strictly-larger` ‚Äî the kernel of `avg ‚àò step`
  strictly contains both the step-then-`avg` loss and `ker step`
  alone. Witness `g6 = [1, -1, -1, 1]` at `n = 4`, block size 2
  (`step-then-avg = avg {1} {2} ‚àò step`): `g6` lies in
  `ker (avg ‚àò step)`, is not a scalar multiple of `nyquist 2`, and
  has `step g6 ‚âÝ 0`.

== Infrastructure

The `step`-specific algebra below stays local to this file. The one
genuinely reusable Vec-rotation fact has been *extracted* to the
top-level `VecRotation` module (polymorphic over any carrier with an
involutive negation): `rotL-alternating` / `rotR-alternating`, plus
the `map-alternating` corollary. That module is what closes gap (a).

* `step-homog` ‚Äî `step (map (c *_) v) ‚â° map (c *_) (step v)`
  (‚Ñö-scalar homogeneity of `step`).
* `step-additive` ‚Äî `step (v + w) ‚â° step v + step w` (additivity).
  Together with `step-homog` this is the ‚Ñö-linearity of `step`.
* `StepKernelSubspace` ‚Äî record bundling `closed-+` and `closed-*‚Çó`;
  with the `StepKernel` predicate and witness `step-kernel-subspace`,
  it packages the kernel as a ‚Ñö-subspace.

Eight Vec-over-‚Ñö lemmas, reusable for future Vec-valued-field work:

[cols="1,3", options="header"]
|===
| Lemma | One-line statement

| `zip-interchange`  | `(A+B)+(C+D) ‚â° (A+C)+(B+D)` pointwise on vectors
| `zip-sub-exchange` | `(A+C)-(B+D) ‚â° (A-B)+(C-D)` pointwise on vectors
| `++‚à∑ ≥`             | `zipWith _+_` distributes over snoc `_‚à∑ ≥_`
| `reverse-zipWith`  | `reverse` distributes over `zipWith _+_`
| `add-rotL`         | left rotation distributes over `zipWith _+_`
| `add-rotR`         | right rotation distributes over `zipWith _+_`
| `scale-+`          | `map (c *_)` distributes over `zipWith _+_`
| `scale-zero`       | `map (c *_)` annihilates the zero vector
|===

== Former gaps (both closed)

=== (a) `‚àÄ n` generalisation of `nyquist-in-step-kernel` ‚Äî CLOSED

The general `nyquist-in-step-kernel : ‚àÄ n ‚Üí step (nyquist n) ‚â°
replicate (2¬∑n) 0‚Ñö` now lands (no postulate, no size restriction).
The Vec-rotation fact stdlib does not provide ‚Äî
`rotL (nyquist n) ‚â° map -_ (nyquist n)`, and the `rotR` twin ‚Äî
was extracted to the `VecRotation` module as the polymorphic
`rotL-alternating` / `rotR-alternating` (each requiring an
involution hypothesis, discharged for ‚Ñö by the ring solver). The
in-file wiring (Rung 5a (closed)) bridges `step`'s local rotations
to the module, lifts the per-entry Nyquist eigenvalue-0 identity
pointwise, and transports across the `vcast ‚àò concat ‚àò replicate`
builder. `nyquist-in-step-kernel-1` / `-2` are retained as fast
`refl` sanity checks at the smallest sizes.

=== (b) `step-kernel-at-4-is-nyquist-line` (rank-1 characterisation) ‚Äî CLOSED

`step-kernel-at-4-is-nyquist-line : ‚àÄ (f : Field 4) ‚Üí step f ‚â°
replicate 4 0‚Ñö ‚Üí Œ£[ c ‚àà ‚Ñö ] f ‚â° map (c *_) (nyquist 2)` now lands.
The subspace closures (`step-additive`, `step-homog`,
`StepKernelSubspace`) establish that `ker step` is a ‚Ñö-subspace; this
pins *which* subspace ‚Äî the Nyquist line. The rank-1 fact is the 4√ó4
‚Ñö system `step [a,b,c,d] ‚â° 0`, whose only solution is
`[a,‚àía,a,‚àía] = a¬∑(nyquist 2)`. It is discharged with the stdlib ‚Ñö
ring solver (`Data.Rational.Solver`): constant folding does the
linear algebra (4¬∑¬º = 1, ¬Ω¬∑2 = 1), so no cancellation lemma, nonzero
side condition, or matrix module is needed. The containment result
`step-kernel-contains-nyquist-multiples` remains as the easy
direction.

== Why this stays in Gate 3

The example is a concrete worked instance demonstrating existing echo
machinery on a real domain (rational fields over a periodic grid). It
introduces no new bridge axis; it does not claim transport and
coarse-graining are independent decorations producing substantive
cross-axis content (the EI-2 investigation settled that question
negatively); and it does not reach toward Gate 2's
characteristic-theorem-family criteria. The strictly-larger kernel of
`avg ‚àò step` is *structured loss* ‚Äî two distinguishable annihilation
sources composing ‚Äî not bridge-axis interaction.
