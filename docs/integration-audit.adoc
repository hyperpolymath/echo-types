== Quality audit

[#gate-2]
=== Gate 2 (`one.zip`)

==== Strengths

The audit does what audits should do: states a falsifier, applies
it to each nominee, and *reports the negative results*. Two
original nominees were withdrawn after the audit found them
reducible:

* `EchoCharacteristic.visible-constraint` reduces to `proj₂` of the
  underlying Σ once `visible := proj₁` is unfolded. Formalised in
  `VisibleConstraintAudit.agda` as `visible-constraint-is-proj₂`,
  proved by `refl`.
* `EchoIntegration.knowledge-and-controlled-degradation` is `_×_`
  of two facts over disjoint frameworks (the conjuncts are over
  different functions, sharing no data, with the second consuming
  no hypotheses). Formalised in `IntegrationAudit.agda` as
  `N3-is-product` and `split-merging-uses-no-hypotheses`.

These withdrawals are the right call. They also exposed a real hole
in the repository: there was no genuine cross-decoration integration
theorem. Rather than leave the hole, the gate-2 instance built one:
`characteristic.RoleGraded`. The new module:

* Defines `RoleGEcho : Role → Grade → Set` — a 2D family that
  varies non-trivially in both axes (`RoleEcho Client true` vs.
  `RoleEcho Server true` at keep grade).
* Defines two independent actions `applyRole`, `applyGrade`.
* Proves `choreo-grade-commute` — both decoration witnesses appear
  on both sides of the conclusion.

The headline theorem is honest about its limits: 18 cases, 17
trivial collapses to `tt` (residue/forget grades degenerate to ⊤)
or `e` (reflexive cases), with the load-bearing case being
`(c⊑s, keep≤keep)` reducing to `client-to-server e`. The instance
acknowledges this in their own falsifier discussion.

==== Recipe insight (cross-lane impact)

`gate-2-handoff.adoc` Observation G frames all four surviving
nominees as instances of a single recipe: custom propositional
decoration order + echo-indexed family + transport action. This
is _exactly the integration-flavoured heuristic_ from gate-1's
`gate-1-handoff.adoc`. Cross-lane consistency: ✅. The two lanes
converged independently on the same structural insight.

==== Issues to address at integration

. *roadmap-gates.adoc was modified.* Per the parallel-instance
  protocol it was read-only, but the audit's negative findings
  required either updating the nominee table or letting it stand
  in contradiction to the audit. The instance chose to update and
  *flag the change explicitly* in two places. Integrator decision:
  accept the amendment (consistent with the audit), or revert the
  modification and note the discrepancy.
. *Path mismatch.* `roadmap-gates.adoc` names the artefact path as
  `docs/characteristic.adoc`; the audit lives at
  `docs/gate-2-characteristic.adoc`. Flagged in a NOTE at the top
  of the audit doc. Resolution: rename, symlink, or update
  `roadmap-gates.adoc` to match.
. *Stale comment in IntegrationAudit.agda.* The closing comment
  block says "A genuine integration theorem ... is not currently
  present in the repository. Building one is open work" — this
  was true before R4 but is now stale (RoleGraded.agda exists).
  One-line fix.
. *The `.Base` import path is correct* for the user's stdlib 2.x
  but cannot be verified in this audit sandbox (sandbox has 1.7.x).
  Sandbox-patched typecheck (substituting `.Base` paths for the
  1.7.x equivalents) shows the *logic* compiles modulo a
  pre-existing `sym-cong` issue in the repo's own `Echo.agda`
  against 1.7.x. No issues introduced by gate-2.

==== Verdict

Integrate. Apply the three small fixes above. The work demonstrably
improves the gate-2 case from "two original nominees that don't
survive scrutiny" to "four nominees that do, plus a genuine
cross-decoration integration construction".

[#gate-3]
=== Gate 3 (`three.zip`)

==== Strengths

Three examples, each on a different bridge axis (tropical,
epistemic, linear), each with five neighbour-theory tests run
explicitly. The pattern is uniform and disciplined:

. Domain prose with concrete miniature.
. Formalisation summary with actual Agda excerpt.
. (a) forced + (b) bridge-axis + (c) distinctness check.
. Five neighbour-theory comparisons (Galois, quotients,
  refinement types, IFC, provenance), with explicit non-reduction
  arguments and *honest negative results* where the comparison is
  tied (e.g., Σ-relevant refinement types coincide with Echo on
  the type level — flagged as a no-win for echo there, while the
  win against propositional refinement is preserved).
. Explicit falsifiers tied to formal Agda.

Two earlier examples were dropped honestly: `DigestEcho` failed
all three of (a)/(b)/(c); `ChoreographyProjection` failed (c)
because joint-Echo collapse is a pullback-of-fibres property in
any LCCC, not echo-specific. Two further candidates (differential
privacy, lossy compression) were considered and *not attempted*
because faithful formalisation requires probability or real
arithmetic that `--safe` Agda doesn't admit without scaffolding
that would itself need canonicality review. Including them as toys
would test the toy, not the domain. This is exactly the discipline
the prompt asked for.

==== The load-bearing insight: `echo-not-prop`

All three examples carry a constructive proof that the relevant
Echo type is *not a mere proposition*. This is the cleanest
distinctness signal the lane found:

* `TropicalArgmin.echo-not-prop` : `is-prop (Echo cost 5) → ⊥`
* `EpistemicUpdate.echo-not-prop` : `is-prop (Echo obs false) → ⊥`
* `LinearErasure.echo-not-prop` : `is-prop (Echo erase erasedF) → ⊥`

These are formal certificates that propositional-truncation
neighbours (preimage subset, equivalence class, IFC equivalence
relation, propositional-predicate refinement type) lose what Echo
retains. *Cross-lane impact:* this is a stronger distinctness
signal than gate-1 has currently formalised. See
§<<integration,Integration recommendation>> for how to fold it in.

==== Verification

All four files (three examples + `examples.All`) typecheck cleanly
in the audit sandbox (Agda 2.6.3 + agda-stdlib 1.7.x). The
generated `.agdai` files match the instance's claim of clean
verification. Exit code 0 on each.

==== Issues to address at integration

. *Stylistic import inconsistency.* The example files use
  `Data.Product`, `Data.Bool` (1.7.x style) while the user's
  existing `Echo*.agda` modules use `Data.Product.Base`,
  `Data.Bool.Base` (2.x modular style). Both work in 2.x because
  `Data.Product` re-exports from `Data.Product.Base`; they break
  in 1.7.x in the other direction. This is *not a correctness
  issue* — gate-3's files compile in both versions. It is a
  *house-style* issue. The instance flagged it in their handoff:
  "If the maintainer prefers... open import Echo... one-line edit
  per file." Integrator decision; either is fine.
. *Lowercase module path.* `module examples.X where` matches the
  directory `proofs/agda/examples/`. The instance flagged that if
  Pascal-cased directory matching the existing top-level modules
  is preferred, renaming is a one-line edit per file. Integrator
  decision.
. *Self-contained Echo definition.* Each file defines `Echo`
  locally rather than importing the repo's `Echo.agda`. Per the
  brief "imports: only existing Echo*.agda and stdlib" this is a
  technicality that goes the *less-coupled* way; the file uses
  *only* stdlib, no echo modules. Rationale flagged in the
  handoff: "robust to API drift in Echo*.agda during parallel
  work". Acceptable as-is; one-line replacement to use repo's
  `Echo` is trivial if the integrator prefers tight coupling.

==== Verdict

Integrate. The three small adjustments above are stylistic, not
substantive. The work delivers exactly what the prompt asked for:
canonical examples that survive a falsifier-driven bar, with
honest negative results recorded for examples that didn't survive.

== Cross-lane consistency

=== Convergent insights (no conflicts found)

. *The "recipe" pattern.* Gate-2's Observation G (custom
  propositional decoration order + echo-indexed family + transport
  action) matches gate-1's integration-flavoured heuristic
  exactly. The lanes converged on the same structural framing
  independently.
. *The composition axis is not the distinctness signal.* Gate-3's
  dropped `ChoreographyProjection` showed joint-Echo collapse is a
  generic pullback-of-fibres property in any LCCC. Gate-1's
  Sophisticated arguments accordingly do *not* claim composition
  per se as the distinctness signal; they claim the natural 2-cell
  data is Σ-over-preimages-shaped. These are consistent.
. *Type-data vs. propositional truncation.* Gate-3's
  `echo-not-prop` isolates non-truncatability as the cleanest
  distinctness signal. This *strengthens* gate-1: the
  `equalizer→echo-pair` and `meet→echo-intersection` results show
  the 2-cell data is Σ-over-preimages (= echo-shaped); gate-3's
  `echo-not-prop` shows that Σ-over-preimages with multiple
  inhabitants is genuinely non-propositional.

=== Tension worth flagging (not a contradiction)

Gate-2's audit revealed that the existing
`EchoIntegration.knowledge-and-controlled-degradation` is
*not actually a cross-decoration integration theorem*; it is `_×_`
of disjoint facts. Gate-1's `T1-obligation` and `T2-obligation`
stubs in `EchoVsQuotient.agda` and `EchoVsGalois.agda` paraphrase
"EchoIntegration's main theorem" as a cross-decoration integration.

This isn't a contradiction — gate-1's *structural arguments* (that
2-cell data is Σ-over-preimages-shaped in both quotient and Galois
encodings) hold regardless of what specific theorem is being
attacked. But the *target* of the obligation needs adjustment:

* The theorem gate-1 actually needs to attack is now the new
  `characteristic.RoleGraded.choreo-grade-commute` (or whatever
  candidate N5 the integrator settles on), *not* the withdrawn
  `knowledge-and-controlled-degradation`.
* Doc fix: in `gate-1-distinct-phenomenon.adoc`, the references
  to "EchoIntegration's main theorem" should be replaced by
  "the characteristic-family commutation theorem (currently
  `RoleGraded.choreo-grade-commute`)" or similar.
* Code fix: the commented-out `T1-obligation` and `T2-obligation`
  stubs should reference `choreo-grade-commute`'s signature when
  filled in.

This is integration polish, not a structural problem. Gate-1's
arguments, the Miniature/Sophisticated witnesses, and the Joins/
IndexedFamilies sub-cases all stand. They were attacking the
*shape* of an integration theorem; the actual integration theorem
is now `choreo-grade-commute` rather than the paraphrase.

=== Axis count (Q1.4 from gate-1)

Neither gate-2 nor gate-3 surfaced a sixth axis. Both stayed
within the five named axes (choreographic, epistemic, linear,
graded, tropical). Gate-2 explicitly reasoned about a four-axis
breadth in their nominee table (graded, choreo, linear, plus the
Role×Grade product); gate-3 covers three axes in their three
examples. Gate-1's "five axes" claim stands.

=== Neighbour theory list

Gate-1 uses: Galois, quotients, IFC, refinements, provenance.
Gate-3 uses: identical list (with modal type theories added as a
sixth neighbour for the LinearErasure example only). The match is
exact for the five core neighbours.

Gate-3 introduces *modal type theories* as a sixth neighbour for
LinearErasure specifically. Per gate-1's handoff: "If gate-3 finds
a neighbour theory not on this list that subsumes echo-types-as-
applied, it should flag back". Gate-3 *did* flag back: the
neighbour is "co-extensive with" echo, not "subsumes". The
integrator should add modal type theories to the canonical
neighbour list for completeness, with the noted caveat that it's
co-extensive in QTT-class systems.

== Self-audit note

the gate-1 work being audited. To mitigate self-favouritism, gate-1
output was checked against the same lane-discipline and
quality-bar criteria applied to the parallel work. Findings:

  generous to the existing module — the actual existing theorem is

  the retarget is not a structural revision.

  redundant with gate-1's work.

The audit recommends integration of all four bundles with the
adjustments noted. No bundle should be rejected outright (apart
from `two.zip`, superseded by `three.zip`).
